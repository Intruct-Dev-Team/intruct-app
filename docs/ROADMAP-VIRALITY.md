# Roadmap: развитие Intruct с фокусом на виральность курсов

Дата: 2026-01-27

Этот документ описывает план развития Intruct на 3 горизонта (короткий/средний/долгий) с главным фокусом: **виральность курсов** (organic growth), при сохранении качества обучения и безопасности.

## 1) Контекст (что есть сейчас)

Сейчас в приложении уже заложены ключевые части продукта:

- Создание курса (3 шага) из прикреплённых материалов, затем генерация курса.
- Личный список курсов, страница курса/уроков, прогресс и локальные оптимистичные обновления.
- Каталог как отдельный таб (публичная витрина — потенциал для роста).
- Общая архитектура: единый data layer в `services/api.ts`, UI на Tamagui, Expo Router.

Это хорошая база: чтобы курс становился “единицей распространения”, нам нужно добавить **публичные точки входа**, **шеринг**, **deep links**, и **growth loops**, которые естественно возникают в процессе обучения.

## 2) North Star и метрики виральности

### North Star

- **Кол-во уникальных новых пользователей, пришедших через курс-шэринг** (или через публичный каталог) и дошедших до первого урока.

### Core метрики (уровень продукта)

- **K-factor (приближенно)**: приглашения/шеры на пользователя × конверсия в установку/открытие.
- **Share rate**: доля пользователей, которые поделились курсом хотя бы 1 раз.
- **Invite conversion**: переход → установка/открытие → регистрация → старт урока.
- **Activation from share**: % “пришёл по ссылке → начал урок”.
- **Retention D1/D7/D30** для пришедших по шеру.

### Контент-метрики (качество и удержание)

- Completion rate по курсу и урокам.
- Среднее время до “первого вау” (первый полезный урок/квиз).
- Оценка курса/урока (если введём рейтинги).

## 3) Growth loops (виральные циклы) — что именно будет “разноситься”

Ниже — приоритетные циклы, которые можно строить независимо.

### Loop A: “Поделись результатом/прогрессом”

- Триггер: завершение урока/курса.
- Действие: “Поделиться карточкой прогресса/сертификатом”.
- Получатель: видит красивую карточку + ссылку на курс.

Почему работает: пользователю приятно делиться достижением, а получатель получает конкретный полезный объект (курс).

### Loop B: “Поделись курсом как полезной шпаргалкой”

- Триггер: пользователь открыл курс, увидел структуру/план.
- Действие: “Поделиться курсом” (с превью).
- Получатель: открывает курс (deep link), может сохранить себе.

### Loop C: “Курс как публичная страница/витрина”

- Триггер: автор публикует курс.
- Действие: публичная ссылка + веб/preview (опционально).
- Получатель: может открыть курс без регистрации (или с лёгким paywall/soft-gate) и позже зарегистрироваться.

### Loop D: “Ремикс/форк курса”

- Триггер: курс полезный, но хочется “под себя”.
- Действие: “Сделать копию/ремикс” (с указанием автора-источника).
- Получатель: получает курс, автор оригинала получает распространение.

## 4) Продуктовые инициативы (по приоритетам)

### P0 (1–2 недели): минимальная виральность без больших перестроек

1. **Share course**
   - Кнопка “Поделиться курсом” на странице курса.
   - Генерация “share URL” вида: `https://intruct.app/c/<courseId>`.
   - Поддержка deep link в приложении: открытие курса по ссылке.

2. **Share progress**
   - На завершении урока/курса: CTA “Поделиться прогрессом”.
   - Первая версия: шарим ссылку + короткий текст.

3. **Превью-карточка (Open Graph) для ссылок**
   - Чтобы в мессенджерах красиво отображалась карточка курса.
   - Требует поддержки на стороне веб/серверной страницы.

4. **UTM/атрибуция**
   - Маркируем ссылки источником: `?ref=share&src=lesson_finish`.
   - Внутри приложения сохраняем “первый источник” в профиле.

### P1 (2–6 недель): публичность, сохранение и социальное доказательство

1. **Публикация курса (public/private)**
   - В настройках курса: “Опубликовать в каталог”.
   - Поля: язык, категория, обложка, теги.

2. **Сохранить курс себе (Save / Add to My Courses)**
   - При открытии курса по ссылке: CTA “Сохранить” и “Начать”.

3. **Рейтинги и отзывы (минимально)**
   - 5 звёзд + короткий отзыв.
   - Порог/модерация (антиспам).

4. **Авторские страницы**
   - Публичный профайл автора: список опубликованных курсов.

### P2 (6–12 недель): ремиксы, коллаборация, челленджи

1. **Remix / Fork**
   - “Сделать копию и отредактировать” (переименование, добавление материалов).
   - Атрибуция исходного курса.

2. **Групповое прохождение (cohort)**
   - “Проходить вместе” → создаём группу/код.
   - Роли: владелец/участник.

3. **Челленджи и стрики**
   - Еженедельные челленджи: “7 дней — 7 уроков”.
   - Шеринг достижения.

### P3 (3+ месяца): веб-витрина и SEO (если целимся в массовый органик)

1. **Web каталог и страницы курсов**
   - Публичные страницы курса на домене.
   - SEO/индексация, OpenGraph.

2. **Embed карточки курса**
   - Встраиваемый виджет/карточка для блогов/Notion.

## 5) UX/дизайн для шеринга (критично для конверсии)

### Share entry points

- На странице курса: “Поделиться”.
- На экране завершения урока: “Поделиться прогрессом”.
- На экране завершения курса: “Поделиться сертификатом”.

### Share payload

- Текст: короткий и полезный (без “маркетингового шума”).
- Визуал: карточка (изображение) с названием курса, прогрессом и брендингом.
- Ссылка: deep link + UTM.

## 6) Технический план (мобильное приложение)

### Deep links

- Поддержка маршрутов:
  - `/course/:id` (открыть курс)
  - `/course/:id?ref=...` (атрибуция)

### Аналитика событий (минимум)

- `course_share_clicked`
- `course_share_completed`
- `course_link_opened`
- `course_saved_from_share`
- `lesson_finished`

Важно: события должны связываться с источником (ref/utm) и хранить campaign.

### Кэш и производительность

- Не делать лишних refetch на табах (уже начато).
- Дальше: ввести слой кэширования (React Query / аналог) с `staleTime` и умным refetch.

## 7) Backend/Platform требования (высокоуровнево)

- Эндпоинт/механизм для получения публичной информации по курсу (для ссылки).
- Публичная HTML-страница для OpenGraph (или edge-render).
- Сущности:
  - course.visibility (private/public/unlisted)
  - share_event + attribution
  - ratings/reviews (с антиспамом)

## 8) Риски и защитные меры

- Спам/мусорные курсы в каталоге → модерация/репорты/лимиты публикации.
- Дубликаты/ремиксы → атрибуция + ограничение “массового клонирования”.
- Приватность материалов → никогда не публиковать исходные загруженные файлы.
- “Плохой первый опыт” при открытии по ссылке → быстрый preview без регистрации или очень лёгкая регистрация.

## 9) Предлагаемый порядок внедрения (конкретный)

1. Deep link + Share course (P0)
2. Attribution (P0)
3. Публичная страница курса (минимальная) + OG (P0/P1)
4. Save from share (P1)
5. Рейтинги (P1)
6. Remix (P2)
